# Mem0 Route Demo

This example focuses on `mem0` only and demonstrates the three integration
routes supported by `memory/mem0`:

1. `agentic`: agent calls `memory_add` and writes existing memory text.
2. `auto-extractor`: framework extractor writes existing memory text.
3. `auto-ingest`: raw transcript is sent to mem0 native ingest, then mirrored
   back into framework-visible memories.

## Environment

```bash
export OPENAI_API_KEY="your-model-api-key"
export OPENAI_BASE_URL="your-model-base-url"

export MEM0_API_KEY="your-mem0-api-key"
export MEM0_BASE_URL="https://api.mem0.ai"
# Optional:
export MEM0_ORG_ID=""
export MEM0_PROJECT_ID=""
```

## Run

```bash
cd examples/memory/mem0

# 1) agentic + write existing memory
go run . -route agentic

# 2) auto extractor + write existing memory
go run . -route auto-extractor

# 3) auto native ingest + raw transcript to mem0
go run . -route auto-ingest
```

## Useful Flags

```text
-app           mem0 app name
-user          mem0 user id
-session       session id (auto-generated by default)
-model         chat / extractor model name
-wait-timeout  max wait time for memory visibility
```

Each run prints the probe token, assistant output, observed tool calls, and the
memories read back from mem0.
